# Описание
Ядро для управления электронными таблицами, предназначенное для встраивания в сторонние приложения. Взаимодействие с таблицей осуществляется через публичный интерфейс класса Sheet.

# Типы данных ячеек
Реализована полиморфная модель ячеек, использующая идиому PImpl для возможности динамического изменения типа во время исполнения.

+ Пустая (Empty). Создаётся автоматически при появлении ссылки на неё из другой ячейки. Время жизни привязано к наличию зависимостей.

+ Текстовая (Text). Хранит строку. Если содержимое начинается с апострофа ('), он отбрасывается при выводе значения, что позволяет экранировать строки, начинающиеся с символа =. При участии в формульных вычислениях производится попытка преобразования к числу; неудача ведёт к ошибке #VALUE!.

+ Формульная (Formula). Выражение после символа =. Поддерживает динамическое вычисление с учётом ссылок на другие ячейки.

# Управление вычислениями и кэширование
Значения формул кэшируются. При изменении ячеек, входящих в дерево зависимостей, происходит каскадная инвалидация кэша. Для этого поддерживается структура обратных ссылок — каждый узел отслеживает, какие ячейки на него ссылаются, что гарантирует эффективное обновление данных по цепочке.

# Обработка ошибок вычислений
Предусмотрены следующие коды ошибок:

+ #VALUE! — некорректное преобразование типа (например, текст не может стать числом).

+ #REF! — обращение к несуществующей позиции (столбец или строка вне допустимого диапазона).

+ #ARITHM! — арифметическая ошибка (например, деление на ноль).

# Построение синтаксического дерева (AST)
Разбор формул выполняется с помощью генератора парсеров ANTLR 4.13.2. Грамматика описана в файле Formula.g4.

## Поддерживаемые конструкции:

Бинарные арифметические операции (+, -, *, /) с учётом приоритетов.

Унарные операции (знак числа).

Группировка выражений с помощью круглых скобок.

Числовые литералы (целые и вещественные).

Ссылки на ячейки (латинская буква — столбец, число — строка, например B2).
